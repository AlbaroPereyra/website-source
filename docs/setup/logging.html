Ebean uses SLF4J for logging.

SQL and Transaction logging

During development it is expected that you include logging of org.avaje.ebean.SQL and org.avaje.ebean.TXN. These log the sql statements executed, bind values for statement execution and transaction demarcation.

Summary logging

The summary logging of org.avaje.ebean.SUM is useful for showing lazy loading queries and how they relate back to origin queries. This is useful when looking at more complex object graphs are built. In tuning queries for N+1 etc.

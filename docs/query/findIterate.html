<#assign findIterate= "true">

findIterate

Execute the query iterating over the results. This is good for processing large results where we want to avoid holding all the beans/results in memory but instead iterator through them.

close

Note that we must ensure the iterator is closed as it holds JDBC resources.

We should use try with resources or a try finally block to make sure that the QueryIterator is closed regardless of exceptions thrown during processing.

Note that findEach and findEachWhile ensure all resoucres are closed regardless and for this reason they are recommended over findIterate.

There are some use cases where findEach and findEachWhile don't naturally fit and that is when we would look to use findIterate.
